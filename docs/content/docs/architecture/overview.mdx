## PrincГӯpios de Design

### 1. SeparaГ§ГЈo Frontend/Backend
- **Frontend**: SPA React com Vite, deploy estГЎtico (Cloudflare Pages)
- **Backend**: API RESTful serverless (Cloudflare Workers)
- **ComunicaГ§ГЈo**: HTTP JSON com autenticaГ§ГЈo JWT

### 2. Serverless First
- Zero servidores dedicados
- Escala automГЎtica com a demanda
- Cold start mГӯnimo (~5ms no D1)

### 3. Edge Computing
- **D1**: SQLite distribuГӯdo na edge da Cloudflare
- **R2**: Storage distribuГӯdo globalmente
- **Workers**: ExecuГ§ГЈo na edge mais prГіxima do usuГЎrio

## Fluxo de AutenticaГ§ГЈo

1. UsuГЎrio faz login via `POST /api/auth/login`
2. Backend valida credenciais no D1
3. Gera JWT com `TokenService` (24h expiraГ§ГЈo)
4. Frontend armazena token via `authStorage`
5. Cada request inclui `Authorization: Bearer <token>` + `x-tenant-id`
6. Middleware `authMiddleware` valida token em cada rota
