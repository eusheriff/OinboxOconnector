## Princípios de Design

### 1. Separação Frontend/Backend

- **Frontend**: SPA React com Vite, deploy estático (Cloudflare Pages)
- **Backend**: API RESTful serverless (Cloudflare Workers)
- **Comunicação**: HTTP JSON com autenticação JWT

### 2. Serverless First

- Zero servidores dedicados
- Escala automática com a demanda
- Cold start mínimo (~5ms no D1)

### 3. Edge Computing

- **D1**: SQLite distribuído na edge da Cloudflare
- **R2**: Storage distribuído globalmente
- **Workers**: Execução na edge mAutomations próxima do usuário

## Fluxo de Autenticação

1. Usuário faz login via `POST /api/auth/login`
2. Backend valida credenciAutomations no D1
3. Gera JWT com `TokenService` (24h expiração)
4. Frontend armazena token via `authStorage`
5. Cada request inclui `Authorization: Bearer <token>` + `x-tenant-id`
6. Middleware `authMiddleware` valida token em cada rota

## Multi-Tenant
