# Banco de Dados

O Oconnector utiliza **Cloudflare D1**, um banco SQLite distribuído na edge da Cloudflare.

## Tabelas PrincipAutomations

### tenants

\`\`\`sql
CREATE TABLE tenants (
id TEXT PRIMARY KEY,
name TEXT NOT NULL,
owner_name TEXT NOT NULL,
email TEXT UNIQUE NOT NULL,
phone TEXT,
plan TEXT DEFAULT 'basic',
status TEXT DEFAULT 'trial',
joined_at DATETIME DEFAULT CURRENT_TIMESTAMP,
trial_ends_at DATETIME,
stripe_customer_id TEXT,
stripe_subscription_id TEXT,
mrr REAL DEFAULT 0
);
\`\`\`

### users

\`\`\`sql
CREATE TABLE users (
id TEXT PRIMARY KEY,
tenant_id TEXT REFERENCES tenants(id),
name TEXT NOT NULL,
email TEXT UNIQUE NOT NULL,
password_hash TEXT NOT NULL,
role TEXT DEFAULT 'user',
created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
\`\`\`

### clients

## Padrões de Query

### Paginação

Todas as queries de listagem usam LIMIT/OFFSET:

\`\`\`sql
SELECT id, name, email, status FROM clients
WHERE tenant_id = ?
ORDER BY created_at DESC
LIMIT ? OFFSET ?
\`\`\`

### Contagem

\`\`\`sql
SELECT COUNT(\*) as total FROM clients WHERE tenant_id = ?
\`\`\`

### Batch Operations

\`\`\`typescript
const batch = leads.map(l =>
stmt.bind(uuid(), l.id, l.score)
);
await env.DB.batch(batch);
\`\`\`

## Migrações

\`\`\`bash

# Criar nova migração

cp backend/migrations/001_initial.sql backend/migrations/002_new_feature.sql

# Executar localmente

wrangler d1 execute Oconnector-db --file=./backend/migrations/002_new_feature.sql --local

# Executar em produção

wrangler d1 execute Oconnector-db --file=./backend/migrations/002_new_feature.sql --remote
\`\`\`
